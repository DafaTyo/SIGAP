# 📌 Module Task Tracker: Data Migrations (data/migrations)

## 🎯 Core Objective , Responsibility
- Menyimpan **script migrasi** SQL atau Alembic untuk meng‑evolve skema database PostgreSQL.
- Setiap migrasi harus **reversible** (down) dan memiliki nama yang deskriptif.

## 📋 Development Checklist
- [ ] **Create Alembic env** – `env.py` (generated by `alembic init`).
- [ ] **Initial Migration** – `V001_create_tables.sql`
  - Membuat tabel `vendor`, `distribution_report`, `complaint`, `audit_logs`, `metadata_catalog`.
  - Menambahkan extension `pgcrypto`, `postgis`.
- [ ] **Add RLS Policies** – `V002_add_rls_policies.sql`
  - Definisikan `CREATE POLICY` untuk tiap tabel berdasarkan `app.current_user_id` dan `app.current_scope`.
- [ ] **Add OPA Policy Sync** – `V003_opa_sync.sql` (optional placeholder).
- [ ] **Write Migrations README** – cara menjalankan `alembic upgrade head`, rollback, dan checklist untuk CI.

## 🔒 Constraints & Best Practices
- **Naming:** `V<NNN>_<description>.sql` (zero‑padded). 
- **Transactional:** setiap file meng‑gunakan `BEGIN; ... COMMIT;`.
- **Testing:** jalankan migrations di CI pada database temporer, pastikan tidak error.
- **Idempotent:** jangan hard‑code data yang dapat berubah (gunakan `INSERT ... ON CONFLICT DO NOTHING`).
